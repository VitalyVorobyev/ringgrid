# Migrating from 0.8 to 0.9

Release 0.9 removes the last legacy compatibility facade. Migration is mechanical.

## `BoardLayout` / `BoardMarker` removed

The `BoardLayout` / `BoardMarker` types that 0.8 kept as a deprecated shim are
**removed** in 0.9 (Rust and Python). Use the compositional `TargetLayout`:

| 0.8 (deprecated) | 0.9 |
|---|---|
| `BoardLayout::default()` | `TargetLayout::default_hex()` |
| `Detector::from_board(board)` | `Detector::new(target)` / `Detector::from_target(...)` |
| Python `ringgrid.BoardLayout(...)` | `ringgrid.TargetLayout.coded_hex(...)` / `.from_json(...)` |

Build targets with `TargetLayout::coded_hex(...)`, `::rect_24x24()`,
`::with_auto_fiducials(...)`, or the compositional `TargetLayout::new(...)`.

## Legacy `board_spec.json` (v4) still loads

You do **not** need to convert existing target files. `TargetLayout::from_json_*`
(Rust) and `ringgrid.TargetLayout.from_json(...)` (Python) auto-migrate the legacy
v4 `board_spec.json` schema to the canonical v5 `target_spec.json` on load. New
targets are written as v5.

## Target generation is a subcommand family

`gen-target` groups its variants: `gen-target hex | rect | preset | from-spec`.
(As of the CLI split, the published `ringgrid` binary uses a recipe-driven
`ringgrid gen <recipe>` instead — see the CLI Guide.)
